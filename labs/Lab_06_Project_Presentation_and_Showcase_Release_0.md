# Lab 06 - Project Presentation and Showcase (Release 0)

## Lab Type

Showcase lab (execution and evidence only, no new teaching content).

Deployment scope for this showcase is local execution of the integrated Release 0 application using Docker Compose. Cloud deployment is out of scope for Release 0.

## Goal

Prepare and present the Release 0 integrated Agentic AI software application using five evidence trees:

1. Report Tree
2. Working Software Tree
3. Agentic AI Workflow Tree
4. DevOps and CI/CD Tree
5. Contribution Tree

Each student must present their assigned frontend, backend/API, and SQLite database microservice contribution as part of the integrated team application.

## Plan

### Step 1 - Technical Report

Objective: prepare the assessor-ready Release 0 technical report.

Required report sections:

- Project overview
- Repository structure
- Individual software architecture
- Integrated software architecture
- Docker Compose architecture
- DevOps pipeline architecture
- Agentic AI workflow
- GitHub Actions workflows
- Implementation summary
- GitHub Actions evidence
- Docker Compose evidence
- Integrated application screenshots
- Known issues and limitations
- GitHub commit logs
- Contribution logs
- Attendance checkpoints

### Step 2 - Working Software

Objective: demonstrate the complete integrated Release 0 application.

Required evidence:

- Frontend microservice operational
- Backend/API microservice operational
- SQLite database microservice operational
- CRUD functionality
- AI-Mode integrated using Ollama and an approved LLM
- Docker containerisation
- Docker Compose execution
- Integrated application validation

### Step 3 - Agentic AI Workflow

Objective: demonstrate the Release 0 AI workflow.

Required evidence:

- AI-Mode configuration
- Ollama runtime configuration
- Approved LLM configuration (Llama, Qwen or DeepSeek)
- Frontend → Backend/API → Ollama → LLM workflow
- Plan → Act → Observe → Adapt workflow

### Step 4 - DevOps and CI/CD

Objective: demonstrate Release 0 automation.

Required evidence:

- Shared GitHub repository
- student-1.yml to student-5.yml
- Automated build and validation
- Docker Compose execution
- Successful GitHub Actions workflow execution

### Step 5 - Contribution

Objective: demonstrate individual contributions within the integrated application.

Required evidence:

- Team responsibilities completed
- Individual frontend/backend/database responsibilities
- GitHub commits
- Contribution logs
- Attendance checkpoints

## Showcase Run Order

1. Project overview and team responsibilities.
2. Repository structure and software architectures.
3. Demonstrate the integrated Release 0 application.
4. Demonstrate the Agentic AI workflow.
5. Demonstrate GitHub Actions and Docker Compose.
6. Present GitHub commits and contribution logs.
7. Record assessor feedback and Release 1 action items.
